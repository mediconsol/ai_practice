PRD를 **요청하신 방향(교육 ❌ / 업무 구현 ⭕ / 프롬프트 자산화 ⭕)**으로 **완전히 재정의한 정식 버전**을 캔버스에 새로 작성해두었습니다.

### 📌 지금 PRD의 성격을 한 문장으로 요약하면

> **“AI를 배우는 플랫폼이 아니라,
> 의료인이 자기 업무에 맞는 AI 활용 방식을 직접 만들고 실행하는 플랫폼”**

---

## 이번 PRD에서 달라진 핵심 포인트

### 1️⃣ 교육이 ‘목적’이 아님

* 교육/차시는 **가이드 역할**
* 플랫폼의 본질은 **업무 주제 기반 AI 구현**

### 2️⃣ 프롬프트 = 개인 자산

* 프롬프트 생성 → 저장 → 재사용 → 외부 AI 적용
* 단순 입력창 ❌ → **프롬프트 라이브러리 ⭕**

### 3️⃣ “프로그램을 만든다”는 관점

* 문서 요약기
* 안내문 생성기
* 교육자료 생성기
* 보고서 초안 생성기
  → **모두 ‘AI 업무 프로그램’으로 정의**

### 4️⃣ 실제 AI 도구와의 연결을 전제로 설계

* ChatGPT / Gemini / Claude / NotebookLM
* 플랫폼은 **설계·관리·실행 기반**
* 실행은 내부 or 외부 자유

---

## 지금 PRD + 기존 TRD/ERD의 관계

| 문서        | 역할                     |
| --------- | ---------------------- |
| **새 PRD** | 왜 이 플랫폼이 필요한가 (업무 관점)  |
| TRD       | 어떻게 기술적으로 구현하는가        |
| ERD       | 무엇을 자산으로 쌓는가 (프롬프트·실행) |

→ **셋이 완전히 일관된 상태**입니다.

## 이제 정말 중요한 다음 단계 (전략적으로)

이 PRD 기준에서 다음 중 하나로 가면 됩니다.

### 🔥 1순위 (가장 강력)

**의료인이 만들고 싶어할 ‘AI 업무 프로그램 Top 7 정의’**

* 이게 곧 초기 홈 화면이 됨
* 주제 = 곧 제품 가치

### 2순위

**프롬프트 생성 로직 상세 설계**

* 질문 흐름
* 템플릿 규칙
* 추천 AI 로직

### 3순위

**MVP 화면 시나리오**

* “첫 로그인 → 첫 프롬프트 생성 → 첫 실행”
